AI Knowledge Base Skill

專為 AI Agent 設計的長短期記憶整合系統

Deep Research Report • 2026.04.06
02

當前挑戰：記憶與成本的權衡

  • Token 膨脹：知識庫越大，每次讀取的成本越高，最終導致 Context 溢出。
  • 知識斷層：AI 往往只能處理當前視窗，無法有效與舊筆記建立關聯。
  • 整理摩擦：手動建立雙向連結對於開發者而言過於耗時且違反「Vibe Coding」。
03

核心技術：註冊表檢索 (Registry)

這是一項旨在極大化 Token 效率的設計哲學：

  • 輕量索引：維護一個 JSON 檔案，儲存標籤 (Tags) 與核心概念。
  • 精準命中：僅當 2 個標籤或 1 個概念重疊時，Agent 才會讀取全文。
  • 線性縮放：即便擁有 200 則筆記，檢索成本依然維持在低水位。
04

與 Obsidian 深度整合

自動化建構

  • 自動生成雙向連結 [[link]]
  • 標準化 YAML Frontmatter
  • 自動更新 _index.md 索引

智能關聯

  • 基於語意標籤的自動分類
  • 發現跨檔案的潛在連結
  • 實現「認知累積」而非單純紀錄

06

協同效應：Frontend Slides

知識不應止於筆記，應轉化為展示：

  • Show, Don't Tell：自動將研究結果轉化為 HTML 簡報。
  • 視覺預覽機制：在生成前提供多樣化的風格挑選。
  • 零依賴部署：單一 HTML 檔案，支援 Vercel 一鍵發佈。
